Exporting protocol header

You can choose to export a single protocol header to a Microsoft Excel file, or one or more protocol headers to a CS Data file in Protocol header dialog.

Exporting to a Microsoft Excel file

Note that an Excel file can contain no more than 64,000 rows of data. If you choose to export a protocol header with more than 64,000 rows of data, the program will provide options to reduce the data size.

To initiate the export, select a protocol header in the protocol header list, click "Export" button and select "To Excel File". If the protocol header contains more than 64,000 rows of data, you will see Export Option dialog shown below. Otherwise, you will see Export dialog directly.

Export Option dialog


In Export Option dialog you can choose to reduce the data size in either of these two ways:

  • Shrink the data: select "Shrink data to less than 64,000". Data size will be reduced by merging adjacent rows. Measurement values in each merged row are calculated by averaging those in the original rows.

  • Shorten the period: select "Select a certain time period". The start time and end time for the record file are displayed. You can select a different start time and/or end time to shorten the period. To select a different start time or end time, click "[null]" to open the calendar to select a new date and, optionally, click "[null]" to change the hour of the date. Only measurement values recorded within the selected period will be exported resulting in smaller data size. If the selected period isn't short enough, you will be prompted to select another start time or end time.

After clicking "OK" button you will see Export dialog. To terminate the export, click "Cancel" button.

The program will notify you of the result of the export. Note that if you enter a filename that already exists in the selected location, this file will be overwritten. If the file is currently opened by another application, the export will fail.

Exporting to a CS Data file

Unlike an Excel file, a CS Data (.csd) file can contain unlimited number of rows of data.

To initiate the export, select one or more protocol headers in the protocol header list, click "Export" button and select "To CS Data File". You will see the same Export dialog as shown above, where you can complete the export in the same way.
